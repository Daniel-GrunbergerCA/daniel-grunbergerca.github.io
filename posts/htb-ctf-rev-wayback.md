# HTB Wayback - Timestamp-Seeded RNG Brute Force

A man named Michael Tanz generated a 20-character alphanumeric+symbols password sometime between 10–11 December 2013 using a tool called `V1`. The challenge is to crack the password and use it to decrypt an AES-CBC encrypted flag.

## Analysing the Password Generator

Reversing `V1` in Ghidra reveals the character set and the seeding logic. The generator builds an alphabet of lowercase + uppercase letters. If symbols and numbers are included, they get appended.

The RNG seed is derived from the current time:

```c
srand(ptVar5->tm-mday * 1000000 +
      ptVar5->tm-min  * 100     +
      ptVar5->tm-sec            +
      ptVar5->tm-hour * 10000   +
      (ptVar5->tm-year + 0x76c) * 0x540be400 +
      (ptVar5->tm-mon + 1) * 100000000);
```

`tm-year` is years since 1900, and `0x76c` is `1900`, so `(tm-year + 0x76c)` is just the full 4-digit year. The password is then generated by calling `rand()` once per character, using the result modulo the alphabet length as an index.

## Brute-Force Strategy

We know:
- Year: 2013, Month: December (12)
- Day: 10 or 11
- Hour, minute, second: unknown - iterate all 86400 × 2 combinations

Since `srand`/`rand` are C library functions, we call them directly from Python via `ctypes` to reproduce the exact same sequence:

```python
import ctypes
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.backends import default-backend
from cryptography.hazmat.primitives import padding

libc = ctypes.CDLL("libc.so.6")

def decrypt-message(encrypted-message, key-str):
    try:
        key = key-str.encode().ljust(32, b'\x00')
        iv = encrypted-message[:16]
        ciphertext = encrypted-message[16:]
        cipher = Cipher(algorithms.AES(key), modes.CBC(iv), backend=default-backend())
        decryptor = cipher.decryptor()
        decrypted-padded = decryptor.update(ciphertext) + decryptor.finalize()
        unpadder = padding.PKCS7(algorithms.AES.block-size).unpadder()
        return unpadder.update(decrypted-padded) + unpadder.finalize()
    except:
        return None

encrypted-bytes = bytes.fromhex(
    'ad24426047b0ffb03b679773664838462a6f00bdcaf0589dd1748e9ed5c56860'
    '1edc87d974894f9dd9b98cc35535145c494eb0af84c8f78d440a033c91c7de62d'
    '506d8cabdc2a10138b95139bbe60e89'
)

lower   = "abcdefghijklmnopqrstuvwxyz"
upper   = "ABCDEFGHIJKLMNOPQRSTUVWXYZ"
symbols = "!@#$%^&*-+"
numbers = "0123456789"
alphabet = lower + upper + symbols + numbers
alpha-len = len(alphabet)

for day in [10, 11]:
    for h in range(24):
        for m in range(60):
            for s in range(60):
                raw-seed = (12 * 100000000) + \
                           (2013 * 0x540be400) + \
                           (day  * 1000000)    + \
                           (h    * 10000)      + \
                           (m    * 100)        + \
                           s
                libc.srand(ctypes.c-uint(raw-seed).value)
                password = "".join([alphabet[libc.rand() % alpha-len] for - in range(20)])
                flag = decrypt-message(encrypted-bytes, password)
                if flag:
                    print(f"[+] Password: {password}")
                    print(f"[+] Flag: {flag.decode()}")
                    exit()
```

Matching `srand`/`rand` behaviour exactly - including the `c-uint` cast to replicate C's unsigned integer overflow - is the key detail. Without it the seed values for late December wrap differently and the search misses.
